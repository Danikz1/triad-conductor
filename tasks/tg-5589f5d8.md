## 1. Обзор проекта

### Цель

Персональный Telegram-бот, выполняющий роль Chief of Staff: управляет календарём, почтой, задачами, контактами, идеями и финансами через единый интерфейс — текст и голосовые сообщения в Telegram.

### Принципы

- **Единая точка входа** — всё через Telegram, никаких других интерфейсов
- **Voice-first** — голосовые сообщения как основной способ ввода
- **Подтверждение перед действием** — бот всегда показывает, что понял, и ждёт “ок” перед записью
- **Notion как база** — все структурированные данные живут в Notion
- **Google как операционный слой** — календарь и почта через Google APIs
- **Claude как мозг** — обработка естественного языка, классификация намерений, генерация структурированных данных

### Пользователь

Один пользователь — Данияр. Бот привязан к одному Telegram-аккаунту. Все интеграции настроены на его Google и Notion аккаунты.

### Intent Engine — классификация намерений

Каждое входящее сообщение (текст или расшифрованное голосовое) отправляется в Claude с системным промптом, который классифицирует намерение:

```python
INTENT_SYSTEM_PROMPT = """
Ты — Intent Engine персонального ассистента. Проанализируй сообщение пользователя 
и определи намерение. Верни JSON:

{
  "intent": "<тип намерения>",
  "confidence": <0.0-1.0>,
  "entities": { <извлечённые сущности> },
  "clarification_needed": "<вопрос, если что-то неясно, иначе null>"
}

Возможные intent:
- "create_meeting" — создать встречу/событие
- "check_calendar" — посмотреть расписание
- "check_email" — проверить почту
- "send_email" — отправить письмо
- "create_contact" — добавить/обновить контакт в CRM
- "create_task" — создать задачу (себе или делегировать)
- "delegate_task" — делегировать задачу конкретному человеку
- "log_idea" — записать идею
- "log_knowledge" — записать в базу знаний
- "accept_report" — принять отчёт
- "check_subscriptions" — посмотреть подписки
- "add_subscription" — добавить подписку
- "upload_bank_statement" — загрузка банковской выгрузки
- "financial_summary" — финансовая сводка
- "morning_briefing" — запросить брифинг
- "followup_check" — проверить follow-up'ы
- "search_contacts" — найти контакт
- "search_knowledge" — найти в базе знаний
- "general_question" — общий вопрос к AI
- "unknown" — не удалось определить

Entities для каждого intent:
- create_meeting: {title, date, time, duration, attendees[], location, notes}
- create_contact: {name, company, role, phone, email, notes, context}
- create_task: {title, description, assignee, deadline, priority}
- delegate_task: {title, description, assignee, deadline, priority}
- log_idea: {title, description, category, related_project}
- log_knowledge: {title, content, category, tags[], source}
- add_subscription: {name, amount, currency, frequency, next_payment, category}

Если дата/время не указаны явно, не выдумывай — поставь null.
Если confidence < 0.7, задай уточняющий вопрос в clarification_needed.

Текущая дата и время: {current_datetime}
Часовой пояс пользователя: Asia/Almaty (UTC+5) / America/Los_Angeles (когда в US)
"""
```

### Confirmation Engine — подтверждения

Каждое действие, изменяющее данные, проходит через подтверждение:

```python
# Пример flow
# 1. Пользователь отправляет голосовое
# 2. Whisper транскрибирует → "Встретился с Каримом из Kaspi, 
#    обсудили интеграцию платежей, он пришлёт КП до пятницы"
# 3. Claude классифицирует → intent: create_contact + create_task (follow-up)
# 4. Бот формирует карточку подтверждения:

"""
📋 Я понял так:

**Новый контакт:**
👤 Карим
🏢 Kaspi
📝 Обсудили интеграцию платежей

**Follow-up задача:**
📌 Карим пришлёт КП 
📅 Дедлайн: пятница, 7 марта
⚡ Приоритет: средний

Всё верно? Отправь ✅ или скажи что поправить.
"""

# 5. Пользователь подтверждает → запись в Notion
# 6. Бот отвечает: "Готово! Контакт добавлен, напомню в пятницу про КП."
```

## 5. Модули и функциональность

### 5.1 Voice Processing Module

**Файл:** `modules/voice.py`

```
Поток обработки голосового сообщения:
1. Получить voice/audio из Telegram
2. Скачать файл (.ogg)
3. Конвертировать в формат для Whisper (если нужно)
4. Отправить в Whisper API → получить текст
5. Отправить текст в Intent Engine (Claude)
6. Роутить по intent
```

**Особенности:**

- Поддержка русского и английского языка (Whisper автоматически определяет)
- Для длинных голосовых (>1 мин): Claude делает краткое резюме перед обработкой
- Если в голосовом несколько намерений (“запиши контакт и создай задачу”), Claude разбивает на отдельные действия

### 5.2 Calendar Module

**Файл:** `modules/calendar.py`

**Функции:**

|Команда            |Действие                                                            |
|-------------------|--------------------------------------------------------------------|
|Создание встречи   |Парсит дату/время/участников → Google Calendar API → создаёт событие|
|Просмотр расписания|“Что у меня сегодня/завтра/на неделе?” → список событий             |
|Перенос/отмена     |“Перенеси встречу с Каримом на час позже” → обновление события      |
|Напоминание        |За 15 мин до встречи → push в Telegram с контекстом из CRM          |
|Конфликт           |Если время занято — предупреждает и предлагает альтернативы         |

**Контекст перед встречами:**

При напоминании о встрече бот автоматически подтягивает из CRM:

```
🔔 Через 15 минут: Встреча с Каримом (Kaspi)

📋 Контекст:
• Должность: Head of Partnerships
• Последняя встреча: 15 февраля — обсуждали интеграцию платежей
• Открытый follow-up: ждём КП от него (дедлайн был 7 марта — просрочен)
• Связанный проект: WhiteRock платёжная инфраструктура
```

### 5.3 Gmail Module

**Файл:** `modules/gmail.py`

**Функции:**

|Команда        |Действие                                                                          |
|---------------|----------------------------------------------------------------------------------|
|Проверка почты |“Что в почте?” → Claude суммаризирует топ-5 непрочитанных                         |
|Поиск          |“Найди письмо от Kaspi за прошлую неделю” → поиск и сводка                        |
|Ответ          |“Ответь Кариму, что мы согласны с условиями” → черновик → подтверждение → отправка|
|Утренняя сводка|Автоматический парсинг важных писем в брифинге                                    |

**Фильтрация приоритетов:**

Claude классифицирует письма:

- 🔴 **Срочные** — от ключевых контактов (список VIP из CRM), содержат слова: urgent, deadline, payment, contract
- 🟡 **Важные** — от бизнес-контактов, подписки на которые активны
- ⚪ **Информационные** — рассылки, нотификации, промо → только в дайджесте

```python
# scripts/google_auth.py
from google_auth_oauthlib.flow import InstalledAppFlow

SCOPES = [
    'https://www.googleapis.com/auth/calendar',
    'https://www.googleapis.com/auth/gmail.modify',
    'https://www.googleapis.com/auth/gmail.send',
]

flow = InstalledAppFlow.from_client_secrets_file('credentials.json', SCOPES)
creds = flow.run_local_server(port=0)

print(f"Refresh token: {creds.refresh_token}")
# Сохранить в .env как GOOGLE_REFRESH_TOKEN
```

## 9. Безопасность

### Критические правила

1. **Только один пользователь** — бот проверяет `TELEGRAM_OWNER_ID` для всех команд. Чужие сообщения игнорируются (кроме отчётов от зарегистрированных исполнителей).
2. **Исполнители** — ограниченный доступ. Могут только:
- Получать уведомления о задачах
- Отправлять отчёты и обновления по задачам
- Подтверждать получение задач (/ack)
- Отмечать задачи выполненными (/done)
1. **API ключи** — только в .env, никогда в коде
2. **Google токены** — хранятся в отдельном volume, не в репозитории
3. **Notion** — Integration имеет доступ только к нужным базам
4. **Подтверждение перед действием** — никакие данные не создаются/изменяются без явного ✅ от пользователя
5. **Логирование** — все действия логируются, но без чувствительных данных (тел API ключей, паролей)
